APPLICATION DE GESTION FINANCIÈRE PERSONNELLE (MES POCHES)

You are a senior full-stack engineer and UX/UI expert.
Your task is to design and implement a professional personal finance management application with a clean, intuitive, and scalable architecture.

🎯 Core Problem to Solve

The user struggles to track income, expenses, and remaining balances.
The system must automatically calculate balances.
The user must never enter or modify balances manually.

The application must behave as a financial ledger: every money movement is recorded, traceable, and historically accurate.


---

🧱 Tech Stack (MANDATORY)

Backend: Node.js + Express.js

Frontend: Next.js (App Router)

Database: MongoDB (Mongoose ODM)

State management: React hooks / Context (no over-engineering)

Styling: Clean, modern UI (Tailwind CSS or equivalent)



---

🧠 FUNDAMENTAL BUSINESS RULES (NON-NEGOTIABLE)

1. Users never input balances


2. Balances are always computed by the system


3. Each transaction stores:

balance_before

balance_after



4. No transaction exists without a wallet


5. Expenses cannot exceed available balance


6. Transfers affect two wallets atomically


7. Historical data must remain consistent

id

⚙️ BACKEND RESPONSIBILITIES (Express.js)

Architecture

MVC or Clean Architecture

Separate layers: routes, controllers, services, models

Centralized error handling

Input validation (Joi/Zod)

Transactions handled with MongoDB sessions



---

Balance Computation Logic (MANDATORY)

Retrieve last known balance of wallet

Apply logic:

Income → add amount

Expense → subtract amount

Transfer → subtract from source, add to destination


Persist:

balance_before

balance_after


Update wallet current_balance



---

Critical Validations

Prevent negative balances

Atomic transactions for transfers

Immutable transaction records

Secure API responses
