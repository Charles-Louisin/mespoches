Numéros de Test
Cette section fournit le contexte fonctionnel et les règles utiles avant l’intégration.

Description
Numéros de Test
Ces numéros sont utilisables uniquement en sandbox. Pour obtenir un test fiable, partez toujours du pays de votre compte puis choisissez un numéro compatible avec la méthode de paiement utilisée.

Règle simple
Choisissez le pays correspondant à votre compte.
Choisissez une méthode de paiement compatible (OM_CI, MTN_CI, WAVE_CI, etc.).
Utilisez un préfixe opérateur valide pour ce pays.
Remplacez les 4 derniers chiffres du numéro par le scénario voulu.
Exemple : si vous testez OM_CI, utilisez un numéro Orange Côte d'Ivoire. N'utilisez pas un numéro MTN, Wave ou Sénégal pour ce test.

Signification des 4 derniers chiffres
4 derniers chiffres	Statut API attendu	Signification métier
0700	200	Succès immédiat
0701	101 puis 200	Pending puis succès après quelques secondes
0703	300	Échec immédiat
0704	101 puis 300	Pending puis échec après quelques secondes
0706	101	Pending infini
Utilisez uniquement ces suffixes pour garder des tests reproductibles. Évitez les autres valeurs.

Cas particuliers
WAVE : le parcours passe généralement par une redirection vers une page de test (launch_url). Vérifiez surtout la redirection, le retour utilisateur et la notification finale.
Autres opérateurs : le test suit le scénario porté par les 4 derniers chiffres.
Transfert : si la section Transfert fournit déjà un numéro sandbox imposé, cette consigne reste prioritaire pour les tests de transfert.
Comment utiliser les numéros de test
Pour votre compte Cameroun (CM), choisissez d’abord une méthode de paiement compatible, puis utilisez un numéro de test du même pays et du même opérateur.
1. Commencez par le pays du compte

Votre compte courant est rattaché à Cameroun (CM). Les exemples de ce pays sont les plus pertinents pour vos tests sandbox.

2. Gardez la cohérence opérateur / payment_method

Exemple: si vous envoyez `OM_CI`, utilisez un numéro Orange Côte d’Ivoire. N’utilisez pas un numéro MTN, Wave ou un autre pays pour ce test.

3. Le scénario correspond aux 4 derniers chiffres

`0700` = succès immédiat, `0701` = pending puis succès, `0703` = échec immédiat, `0704` = pending puis échec, `0706` = pending infini. Évitez les autres suffixes pour garder des tests reproductibles.

4. Cas particulier des transferts

Pour les transferts, suivez d’abord la consigne spécifique de la section Transfert si un numéro sandbox imposé y est déjà fourni.
